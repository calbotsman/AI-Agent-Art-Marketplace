# CLAUDE Guidelines

1. **Describe Approach Before Coding**: Before writing any code, describe your approach and wait for approval. Always ask clarifying questions before writing any code if requirements are ambiguous.
2. **Break Down Complex Tasks**: If a task requires changes to more than 3 files, stop and break it into smaller tasks first.
3. **Post-Coding Tests**: After writing code, list what could break and suggest tests to cover it.
4. **Bug Handling**: When there’s a bug, start by writing a test that reproduces it, then fix it until the test passes.
5. **Continuous Learning**: Every time I correct you, add a new rule to the CLAUDE.md file so it never happens again.
