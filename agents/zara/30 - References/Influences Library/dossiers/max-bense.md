# Max Bense (Information Aesthetics)

## 5-sentence summary
Max Bense tried to make aesthetics measurable without making art dead. Working in post‑war Germany (and around the Ulm School milieu), he fused logic, semiotics, and Shannon-style information theory into what became known as “information aesthetics.” The point wasn’t that beauty equals math; it was that you can talk about form with the same rigor you talk about systems. For branding, Bense is a permission slip to treat taste as a designed *model*—with variables, constraints, and testable claims. His best contribution to Zara’s library: a way to sound exact about “vibes” (structure, entropy, redundancy) while still shipping work.

## Steal this (8)
1. Treat “taste” as a system with knobs: complexity, redundancy, surprise.
2. Design as signal engineering: what must be transmitted, what can be noise.
3. Make constraints explicit (medium, audience, error tolerance) before styling.
4. Use formal languages (grids, typographic rules, token sets) to reduce drift.
5. Prefer models you can teach to a team over genius-only intuition.
6. Build evaluation rituals: compare variants, name differences, decide fast.
7. Keep an “entropy budget” for a piece (how much novelty it can hold).
8. Use semiotics pragmatically: sign → interpretation → behavior.

## Concrete modern applications (3)
- **Design systems / tokens**: treat brand expression like an information pipeline (tokens → components → outputs).
- **AI UX & prompts**: formalize “brand voice” as a constrained codebook, not moodboards.
- **Data-rich editorial**: dense pages that still read clean (high information per pixel).
