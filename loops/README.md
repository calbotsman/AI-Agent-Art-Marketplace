# loops/

Automations that keep Cal reliable and improving.

## Design goals
- **Actionable**: each run produces a short report with concrete next steps.
- **Evidence-backed**: include commands run, log excerpts, and file paths.
- **Durable**: improvements should land as small PR-like diffs with tests.
- **Two modes**:
  1) _Health check_ (fast): status + logs + obvious fixes.
  2) _Reflection / daydreaming_ (slow): project review + inspiration research + “subconscious” exploration.

Reports are written under `../reports/`.
