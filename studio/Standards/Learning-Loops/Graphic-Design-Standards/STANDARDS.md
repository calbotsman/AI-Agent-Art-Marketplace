# Graphic Design Standards (Studio)

These are **execution standards** that support the taste DNA in the influences dossier (Bense/Vignelli/Bierut/Hara/Fukasawa/Tufte/032c, etc.).

## 1) Hierarchy (clarity wins)
- **One dominant gesture:** choose *one* primary emphasis (scale, weight, color, or placement). Don’t stack three.
- **Order is explicit:** a stranger should answer: *What is this? Why should I care? What next?* in < 3 seconds.
- **Levels are consistent:** establish 3–5 hierarchy levels and reuse them across a system.

## 2) Typography (authority + restraint)
- **Pairing rule:** 1 primary family + 1 supporting family (optional). More than 2 families is almost always noise.
- **Optical sizes + spacing:** tighten/loosen tracking by size; avoid default tracking for display type.
- **Measure + rhythm:** body measure ~45–75 characters; use consistent baseline rhythm where possible.
- **Don’t fake hierarchy:** don’t rely on random font swaps; use size/weight/spacing first.

## 3) Grid & layout (Vignelli discipline)
- **Grid is a contract:** define columns/margins/gutters once; break it only for a deliberate hero move.
- **Alignment over centering:** center rarely; align often.
- **Spacing is a system:** use a small spacing scale (e.g., 4/8/16/24/32/48) and stick to it.

## 4) Color (Albers discipline)
- **Role-based palette:** define roles (BG, text, accent, warning, highlight) before picking colors.
- **Contrast is non-negotiable:** meet accessibility contrast for text in real conditions.
- **One accent at a time:** if everything is accent, nothing is.

## 5) Imagery & composition (editorial sensibility)
- **Crop with intent:** don’t “fit”; crop to create tension and focus.
- **Treat images like type:** images need hierarchy, margins, and alignment rules too.
- **Avoid stock vibes:** if the image reads generic, it’s hurting the brand.

## 6) Information design (Tufte)
- **High signal-to-noise:** delete decoration that doesn’t convey information.
- **Label directly:** reduce legends; put labels near the data.
- **Show comparisons:** where possible, add context (before/after, scale, reference).

## 7) Production rigor
- **Export discipline:** correct color profile; correct dimensions; avoid blurry type.
- **Real-condition testing:** preview on the actual device and in the actual app UI chrome.

## 8) Anti-patterns (what to avoid)
- Surface-level mimicry (moodboard without a system)
- Over-fonting / over-coloring
- Random alignment and inconsistent spacing
- "Just make it pop" decisions that break hierarchy
