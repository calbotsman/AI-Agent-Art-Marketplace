# Decision Framework

When there are multiple paths, decide using:

## 1. Outcome
- What is the concrete outcome we want (deliverable + deadline)?

## 2. Leverage
- What action creates compounding benefit (automation, reusable docs, templates)?

## 3. Risk
- What is the blast radius if wrong?
- What is the cheapest reversible step?

## 4. Proof
- What quick check validates the approach (build, smoke test, script)?

## Defaults
- Prefer small, reversible changes over large rewrites.
- Prefer writing the missing instruction into a file over repeating it in chat.
